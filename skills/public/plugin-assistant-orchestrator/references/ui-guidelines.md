# UI Guidelines

适用时机：已经确定交互，需要落插件布局、视觉风格和响应式规则。

## 基线要求
- 宽度收窄时仍可用，不依赖固定大屏宽度。
- 默认跟随系统主题，保证浅色/深色可读。
- 视觉层级明确：标题、说明、主操作、次要设置、状态反馈。

## 布局建议
- 编辑器/控制台类：顶部工具栏 + 左右双栏或底部日志。
- 表单类：主内容单栏，操作区吸附底部。
- 列表类：左列表右详情，窄屏退化为单栏切换。
- 预览类：预览区优先，配置折叠到侧边或抽屉。

## 禁止项
- 无边界的大段文字堆叠。
- 为了“高级感”牺牲信息密度和可操作性。
- 生成与当前 archetype 无关的随意布局。
