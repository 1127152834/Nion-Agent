# Plugin Archetypes

在开始写 UI 和脚手架前，优先从下面 archetype 中选择最接近的一种。

## console-workbench
- 适合：IDE、终端、编译日志、命令执行、代码工作台
- 结构：顶部工具栏 + 主工作区 + 底部或侧边日志
- 强项：高信息密度、频繁刷新、状态可追踪

## form-panel
- 适合：配置生成器、参数面板、导出工具、发布表单
- 结构：说明区 + 分组表单 + 固定底部操作条
- 强项：明确输入输出、易于发布前检查

## list-detail
- 适合：资源浏览、规则管理、批量处理、文件导航
- 结构：左侧列表 / 树，右侧详情或操作面板
- 强项：多对象切换与上下文保持

## preview-inspector
- 适合：实时预览、可视化检查、截图比对、结果验收
- 结构：预览主体 + 检查面板 + 快速刷新
- 强项：边看边调，适合调试闭环
