# Interaction Playbook

适用时机：需求已基本清楚，需要收敛插件入口、核心动作和文件匹配策略。

## 必须明确的三件事
- 入口点：文件、目录、项目，还是全部内容。
- 核心动作：打开、预览、保存、转换、生成、批量操作等。
- 文件匹配：扩展名、路径模式、项目标记。

## 推荐输出
- 方案 A：最快上线，动作少，学习成本低。
- 方案 B：功能更完整，适合频繁使用。
- 方案 C：只在复杂场景下提供，默认不推荐。

## 决策原则
- 插件不是完整应用，优先最短任务路径。
- 一次主流程不要超过 3 个关键动作。
- 如果目标文件范围不清，先收窄到最稳定的文件类型。
