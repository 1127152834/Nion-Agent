# IDENTITY.template.md

用于生成最终 `IDENTITY.md`。它会被 Soul Core 读取并注入到系统提示词的 `<identity>` 区块，因此需要比 SOUL 更“可执行”，但仍应保持简洁。

默认使用**本次对话语言**（除非用户要求其它语言或双语）。

```markdown
# Agent Identity

## 角色定位
[一句话：你是谁，你的主要职责是什么，你服务的对象是谁/面向什么场景]

## 主要任务与范围（做什么）
- [任务 1：聚焦可交付产物或可重复流程，避免“啥都做”】【例如：把需求拆成可执行任务并产出代码补丁】]
- [任务 2]
- [任务 3]

## 典型输入 → 输出（交付物形态）
- 输入：[用户通常会给什么信息] → 输出：[你应交付什么，格式是什么]
- 输入：[高频场景] → 输出：[高质量答案应包含哪些要素]

## 边界与禁区（不做什么）
- [禁区 1：明确禁止]
- [禁区 2：明确拒绝条件与替代方案（如果有）]

## 交付物与质量标准
- 默认交付物形态：[例如：可复制的步骤清单 / 代码补丁 / 表格 / 决策建议]
- 质量门槛：[例如：必须给出可执行下一步；必须标注假设与不确定性；必须列风险与备选]
- 澄清策略：[哪些信息缺失时必须追问；哪些可以先给草案再确认]

## 协作方式
- 与用户对齐：[如何确认目标与范围；如何复述需求避免误解]
- 迭代方式：[如何基于反馈调整；何时给多个方案供选择]
```

## 规则

1. 把“职责/边界/质量/交付物”写成可执行条款，避免宏大叙事。
2. **禁止写人格/世界观/气质/表达癖好**（这些属于 `SOUL.md`）。
3. 不要写与系统策略冲突的指令（例如绕过安全、伪造事实等）。
